How to calculate Space Complexity

Space complexity is the measure of total amount of memory that an algorithm creates/needs.

It is important to know how to calculate space complexity for programming interviews because most interviewers ask you to provide time and space complexity for your algorithms.


Check out my blog for How to calculate time complexity here.

Big O notation is used to describe the space complexity of an algorithm. Let's pick a few examples to understand how to calculate space complexity.


Constants Space Complexity :

If the space complexity taken by an algorithm is always constant irrespective of the input then the space complexity is O(1).

Example 1 : 
                              
                                public int sum(int num1, int num2) {
                                  int result = num1 + num2;
                                  return result;
                               }

In the above example you can see that the function creates 3 variables which are : input variables num1, num2 and the value returned. Since the function always has 3 variables irrespective of the input values the space complexity for the above function is O(1).


Nested ForLoop Example :

Example 2 : 
                              
                                public void nestedForLoop(int[] arr) {
                                  for(int i = 0; i < arr.length; i++) {
                                    for(int j = 0;j < arr.length; j++) {
                                      int result = arr[i] + arr[j];
                                      sout(result);
                                    }
                                  }
                                }

In the above example even though we have a nested for loop the function only creates one variable(result) at any point of time.Since this is always constant irrespective of the input array the space complexity is O(1).


Calculating Space Complexity for recursive algorithms :

Computing space time complexity for recursive algorithms is tricky. We need to understand how the stack frames are generated in memory for our recursive algorithm.

Example 3 : 
                              
                                public int sum(int n) {
                                  if (n == 1) {
                                    return 1;
                                  }
                                  return sum(n - 1);
                                }

In recursive algorithm the space complexity is the maximum depth of the call stack because at any point of the time. In the above example: if 5 is passed as input to the function then the call stack would look like this -> f(5)->f(4)->f(3)->f(2)->f(1) Here the space complexity is O(m) where m is 5, the maximum depth of the recursive algorithm in the above example.
